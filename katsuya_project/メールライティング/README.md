# メールライティング運用（Katsuya プロジェクト）

全案件で揃える**形式**と、案件・ブランドごとの**マインドセット／フッター**の置き場を整理するためのインデックスです。

## Cursor ルール（正本）

| 種別 | パス |
|------|------|
| **形式・フォーマット（全案件共通）** | `.cursor/rules/email-writing-format.mdc` |
| **ひとびじ（藤永・ハウスメール含む）** | `.cursor/rules/hitobizi-lab.mdc` |
| **本田有紀華さん（ハウス・フッター）** | `.cursor/rules/honda-yukika-housemail.mdc`、`.cursor/rules/honda-yukika-mail-footer.mdc` |
| **藤永としての応答方針** | `.cursor/rules/fujinaga-katsuya-persona.mdc`（メール形式は上の共通ファイルを参照） |

個別ルールがあるときは**常に個別を優先**する（`email-writing-format.mdc` 冒頭の優先順位と同じ）。

## Obsidian（下書き・推敲）

| パス | 用途 |
|------|------|
| **`obsidian/メール原稿/たたき台/`** | 案件横断のたたき台・短いメモ |
| **`obsidian/メール原稿/案件メモ/`** | 案件ごとの下書き・メモ（必要に応じて `product/案件/` と同名のサブフォルダを作る） |
| **`ひとびじ/ハウスメール/`** | 藤永さんハウスメールの正本（`katsuya_project/ひとびじ/` 経由） |
| **`本田有紀華コンテンツ/`** | 本田さん系 LP・レター・メールまわり |

詳細は **`obsidian/メール原稿/README.md`**。

## 配信確定・案件ストック（Git 正本）

| パス | 用途 |
|------|------|
| **`product/案件/<案件名>/メール/`** など | 配信に使う確定稿を残す場合の置き場（案件 README に従う） |
| **`product/講師横断/本田有紀華さん案件/`** | 本田さんメールフッター正本（`honda-yukika-mail-footer.md`） |

## 新規案件でやること（チェックリスト）

1. **形式**は共通ルール（`email-writing-format.mdc`）に合わせる。  
2. **マインドセット・フッター**が共通と違うなら、`.cursor/rules/<案件用>.mdc` を追加するか、案件フォルダの `README.md` に明記する。  
3. 下書きは **`メール原稿/案件メモ/<案件名>/`** か **`product/案件/<案件名>/`** 側の運用に合わせる。
