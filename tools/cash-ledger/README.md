# お金のきろく（cash-ledger）

複数口座の入出金・残高をブラウザの **localStorage** に保存するツールです。

## Googleドライブ自動連携（任意）

ブラウザだけで **OAuth 2.0（PKCE）** と **Drive API** を使い、`cash-ledger-sync.json` を読み書きできます。

- **スコープ**: `drive.file`（このアプリが作成したファイル／ユーザーがピッカーで選んだファイルのみ。Drive全体には見えません）
- **Client ID** は「ひとびじ公開」と「自分だけ」の両方に対応できるよう、**ブラウザの localStorage に保存**する方式です（リポジトリに秘密を書かなくて済む）。

### 1. Google Cloud での準備

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成（または既存を選択）
2. **API とサービス → ライブラリ** で **Google Drive API** を **有効化**
3. **API とサービス → OAuth 同意画面**
   - ユーザータイプは用途に合わせて（個人利用なら「外部」でもテストユーザーに自分の Gmail を追加可能）
4. **API とサービス → 認証情報 → 認証情報を作成 → OAuth クライアント ID**
   - アプリケーションの種類: **ウェブアプリケーション**
   - **承認済みの JavaScript 生成元** に、ツールを開く **オリジン** を追加  
     - 例（GitHub Pages）: `https://katsuya-fujinaga.github.io`  
     - ローカル検証: `http://localhost:5500` など**実際に使うオリジンと完全一致**
   - （リダイレクト URI は GIS のトークンクライアント利用時は基本不要なことが多いです）

作成後に表示される **クライアント ID**（`….apps.googleusercontent.com`）をコピーします。

### 2. ツール側の設定

1. ページで **「Drive設定」** を開く  
2. **OAuth 2.0 クライアント ID** に貼り付けて **「設定を保存」**  
3. **「Driveへ保存」** を一度押す（Google の許可ダイアログが出たら承認）  
   - 初回は Drive 上に `cash-ledger-sync.json` が作成され、このブラウザに **ファイル ID** が記録されます。

オプション:

- **自動で Drive に保存**: 入力が落ち着いてから約 3 秒後にアップロード（編集のたびにまとめて送るイメージ）
- **開いたとき Drive が新しければ取り込む**: `_cashLedgerSync.updatedAt` でローカルより新しい場合のみ確認ダイアログのうえで置き換え

### 3. 複数端末（PC・スマホ）

各端末のブラウザで **同じ Client ID** を設定し、**同じ Google アカウント**で許可すれば、同じマイドライブ上のファイルを共有できます。

- **原則**: 同時に両方で編集しない（衝突時は「タイムスタンプが新しい側が優先」になる設計です）
- **トラブル時**: 「Drive設定」内の **ファイルIDをリセット** でローカル記録だけ消せます。次の保存で新規ファイルが増えるので、Drive 上の古い重複は手で整理してください。

### 4. 全員共通の Client ID を HTML に埋める（任意）

チームや公開配布で「入力させず済ませたい」場合のみ、`index.html` の

```html
<meta name="cash-ledger-google-client-id" content="あなたのクライアントID" />
```

に設定できます（**クライアント ID は公開前提**ですが、Cloud Console のオリジン制限は必ず絞ってください）。
