# ひとびじLAB｜ハウスメール

## ルールの正本

`.cursor/rules/hitobizi-lab.mdc` の次を参照する。

- **ハウスメール注意事項**
- **メール共通フォーマット**（改行・サブヘッド・署名・フッター）

## このフォルダ用の要約

| 項目 | 内容 |
|------|------|
| **ファイル名** | **`No.番号_タイトル（スラッグ）_YYYYMMDD.txt`**（**No. → タイトル → 日付**の順。例：`No.2_お金は道具幸せは別皿_20260412.txt`）。通番は**フォルダ運用で決める**（先頭に据えたい号を `No.1` `No.2` にし、**それ以外はファイル名末尾の配信日が古い順**で `No.3` から振る、など）。スラッグが同じでも末尾日付で区別する（例：`No.4_道の作り方_20250319` と `No.7_道の作り方_20260327`）。 |
| **原稿冒頭メタ** | 次の5行＋空行のあとに配信本文（**`本文` という行は置かない**）。メタと区切り線は配信に含めない。**`件名：` 行は置かない**（件名は配信ツールでタイトルと同じにする）。<br><br>`::::::::::::::::::::::::::::`<br>`配信名：藤永勝也｜ひとりビジネスの教科書`<br>`No.◯　YYYY年M月D日配信`<br>`タイトル：◯◯◯`<br>`::::::::::::::::::::::::::::` |
| **連載まわり** | 「前回」「次回」「続き」「1話読切」など、連載・話数のくだりは**書かない**。毎回完結。 |
| **次回予告** | **書かない** |
| **呼称** | **お客さま**に統一（「お客さん」は使わない） |
