# メール原稿の置き場（運用メモ）

## Cursor ルールのレイヤー（メール）

| レイヤー | 置き場 | 内容の目安 |
|----------|--------|------------|
| **形式・フォーマット（全案件共通）** | `.cursor/rules/email-writing-format.mdc` | 冒頭ブロックの順、件名・ファイル名・冒頭メタ、改行とㅤ、ステップの論理、サブヘッド型など |
| **マインドセット・フッター（案件・ブランド別）** | `.cursor/rules/hitobizi-lab.mdc`・`honda-yukika-housemail.mdc`・`honda-yukika-mail-footer.mdc` など | トーン、一人称、読者像、配信名行、署名・フッター全文 |

個別ルールがあるときは**常に個別を優先**する。

**運用ハブ（パス対応の一覧）**：`katsuya_project/メールライティング/README.md`（リポジトリ直下。ボルトからは `katsuya_project/メールライティング/`）。

## フォルダ構成

| サブフォルダ | 用途 |
|--------------|------|
| **`たたき台/`** | 案件に属さない短いメモ・文案の素 |
| **`案件メモ/`** | 案件ごとの下書き（`product/案件/` と同名のサブフォルダを作成） |

## 方針

1. **下書き**はこの `メール原稿/`、または案件ごとのフォルダ（例：`本田有紀華コンテンツ/`）に **`.md` か `.txt`** で置く。  
2. **推敲・リンク・タグ**は Obsidian 上で行う（MyASP に貼る前の「正」はここでよい）。  
3. **配信確定文**は配信ツールに貼る。リポジトリにも残したいときは、同じ内容を **UTF-8 の `.txt`** で保存し、必要なら **`product/案件/<案件名>/メール/`**（`obsidian/product` はリポジトリ直下 `product/` へのリンク）にコピーする（任意）。

## 既存フォルダとの使い分け

- **藤永さんハウスメール**: `ひとびじ/ハウスメール/`（下書き〜完成まで）
- **本田さん系**: `本田有紀華コンテンツ/`（LP・リードメールなど）

迷ったら「ブランドごとに 1 フォルダに寄せる」と検索しやすいです。
